Fork This Archive

CyberCat Sunflower is built as a reusable archive template. The entire data layer is a single JavaScript array of show objects. To create your own artist archive, replace the SHOWS dataset with your own and redeploy on GitHub Pages in minutes.

Show data schema — one entry per show
// Each show follows this structure: { "date": "1977-05-08", // ISO date string "venue": "Barton Hall, Cornell University", "city": "Ithaca, NY", "state": "NY", "setlist": [ "New Minglewood Blues", "Scarlet Begonias", "Fire on the Mountain", "Estimated Prophet" ], "notes": "Often cited as the best GD show ever recorded" }
Deploy your own in 4 steps
# 1. Fork or clone this repo git clone https://github.com/RASvibir/Victor_E._Birkle_III_DEV-Portfolio.git cd Victor_E._Birkle_III_DEV-Portfolio/cybercat-sunflower # 2. Edit the SHOWS array in index.html with your artist's data # 3. Push to your GitHub repo git add . && git commit -m "My custom archive" && git push origin main # 4. Enable GitHub Pages → Settings → Pages → main branch # Your archive goes live at: yourusername.github.io/your-repo/cybercat-sunflower/
